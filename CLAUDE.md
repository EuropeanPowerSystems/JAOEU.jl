# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## What this is

`JAOEU.jl` is a Julia client for the [Joint Allocation Office](https://www.jao.eu) REST APIs (European cross-border electricity capacity allocation: flow-based market coupling on the Core CCR, and explicit auctions via OWSMP). The package was scaffolded with [`OpenAPITemplate.jl`](https://github.com/langestefan/OpenAPITemplate.jl) (a PkgTemplates plugin around `openapi-generator`) on top of an OpenAPI 3.1 spec we synthesize ourselves — JAO publishes neither an OpenAPI spec nor a Postman collection.

The user-facing surface is the *third* layer — named wrappers in `src/conveniences/queries.jl` plus the two platform constructors `PublicationToolClient()` and `OwsmpClient(api_key)`. The codegen layer underneath is rarely the right thing to call directly; reach for it only when no named wrapper exists yet.

## Three layers, stacked

```text
src/
  conveniences/   ← hand-written JAO layer (untouched by codegen)
    queries.jl    ←   named wrappers (net_position, final_domain, monitoring,
                      auction_corridors, auction_horizons, auction_results)
    client.jl     ←   PublicationToolClient + OwsmpClient + the routing
                      pre_request_hook + per-platform `*_api(client)`
                      constructors
    parsing.jl    ←   rows() / total_rows() — unwrap `{data, totalRowsWithFilter}`
    period.jl     ←   to_zoned_utc(DateTime|Date|ZonedDateTime) -> ZonedDateTime UTC
    conveniences.jl ← umbrella include + export list
  client/         ← hand-written template overlay (also untouched by codegen)
    Client.jl     ←   the underlying Client type
    auth.jl       ←   NoAuth/BearerToken/APIKey/BasicAuth + resolve_credentials
    errors.jl     ←   APIError hierarchy + check_response status → typed-exception map
    retry.jl      ←   RetryPolicy + with_retry
    rate_limit.jl ←   TokenBucket + with_rate_limit
    timeout.jl    ←   with_timeout
    middleware.jl ←   DefaultMiddleware + with_defaults
    logging.jl, pagination.jl, show.jl
  api/            ← GENERATED by openapi-generator-cli — DO NOT EDIT
    JAOEUAPI.jl   ←   inner module
    apis/         ←   one file per OpenAPI tag: PublicationToolCoreDAApi (18 ops),
                      OwsmpAuctionsApi (5 ops)
    models/       ←   DataEnvelope, DomainEnvelope, ProblemDetails
gen/
  regenerate.jl   ← rerun codegen against spec/openapi.json
spec/
  openapi.json    ← canonical OpenAPI 3.1 spec consumed by codegen
  openapi.yaml    ← human-readable mirror
scripts/
  build_openapi.jl ← rebuilds spec/openapi.{json,yaml} from a declarative table
```

The generated `src/api/` directory carries `linguist-generated=true` in `.gitattributes`. Re-running `gen/regenerate.jl` nukes and re-creates `src/api/` only — nothing else.

### The one manual edit to a templated file

`src/JAOEU.jl` is templated but `gen/regenerate.jl` does not touch it. We added one line at the end:

```julia
include("conveniences/conveniences.jl")
```

That umbrella include re-exports every name listed in `conveniences/conveniences.jl`'s `export` statements.

## Spec synthesis (no Postman, no Swagger upstream)

JAO publishes **no** machine-readable spec. The Publication Tool runs ASP.NET Core with Swashbuckle disabled, the Utility Tool only exposes SOAP/WSDL, and the OWSMP auctions API ships a 6-page PDF as its sole reference. We consolidate the surface in `scripts/build_openapi.jl` from two sources:

- The open-source [`jao-py`](https://github.com/fboerman/jao-py) client — endpoint paths, parameters, response shapes for the Publication Tool.
- `API_User_Guide_v1.0.pdf` — OWSMP auctions endpoints.

To add or change an endpoint:

```bash
# Edit ENDPOINT_TABLE in scripts/build_openapi.jl
julia scripts/build_openapi.jl      # rewrites spec/openapi.{json,yaml}
julia --project gen/regenerate.jl   # rebuilds src/api/ from spec/openapi.json
```

The script is self-bootstrapping: on first run it activates `~/.julia/environments/jaoeu-scripts` and adds `JSON3`, `YAML`, `OrderedCollections`.

### Synthetic paths and routing

OpenAPI keys operations by `path + method`, but JAO's Publication Tool and OWSMP are physically distinct hosts that we wrap in one spec. We therefore mount every operation under a synthetic prefix:

- `/publicationtool/core/data/<op>` and `/publicationtool/core/system/<op>` → `https://publicationtool.jao.eu/core/api/{data,system}/<op>`
- `/owsmp/<op>` → `https://api.jao.eu/OWSMP/<op>`

The `OpenAPI.Clients.Ctx` constructor concatenates `client.root + resource_path`, so by the time the request is assembled the URL contains the synthetic prefix mid-path. The pre_request_hook in `src/conveniences/client.jl#_routing_hook` strips that prefix from the full URL before the request hits the wire. **If you touch the spec prefixes or the hook, keep that rewrite working — there's an end-to-end smoke test (`monitoring` is the easiest, since it's anonymous and returns a small payload) you can use to verify.**

## Two platforms, two clients

The two API families have different auth and different conventions:

```julia
using JAOEU
using Dates

# Publication Tool — anonymous (a Guest bearer is baked into the SPA;
# pass an explicit bearer only if you have an elevated role token).
client = PublicationToolClient()
env, _ = monitoring(client, DateTime("2024-09-01T00:00"),
                            DateTime("2024-09-02T00:00"))
records = rows(env)   # Vector of JSON.Object — pass to DataFrame, StructArray, …

# OWSMP — API key required (helpdesk@jao.eu, subject `[API] Token Request`)
auct = OwsmpClient(ENV["JAO_OWSMP_API_KEY"])
corridors, _ = auction_corridors(auct)
```

The two clients are the same `Client` struct underneath — the difference is the base URL and the `Auth` strategy installed in their `pre_request_hook`. Don't try to share a client across platforms.

### `Parsed` vs `Raw` — not currently used

Unlike `ENTSOE.jl`, JAO responses are JSON, deserialized by the codegen into `DataEnvelope`/`DomainEnvelope` automatically. There's no separate XML parsing pass, so no `ResponseFormat` dispatch is needed for the named wrappers. If you ever add a wrapper that wants to expose the raw JSON, mirror ENTSOE.jl's `Parsed`/`Raw` pattern.

### Pagination — two-phase pattern for domain queries

`finalComputation`, `preFinalComputation`, `initialComputation` are paginated via `Skip`/`Take`. Idiomatic use mirrors jao-py:

```julia
api = publicationtool_core_da_api(client)
# Phase 1: probe row count
env, _ = JAOEU.JAOEUAPI.publicationtool_core_da_final_computation(
    api, to_zoned_utc(t0), to_zoned_utc(t1); take=0)
total = total_rows(env)
# Phase 2: page through with take=5000 (jao-py's chunk size)
```

Implement a `query_split` style helper in `conveniences/` if you need cross-MTU pulls.

### When you must call the generated layer directly

For operations without a named wrapper, drop into `JAOEU.JAOEUAPI`:

```julia
api = publicationtool_core_da_api(client)
env, resp = JAOEU.JAOEUAPI.publicationtool_core_da_shadow_prices(
    api, to_zoned_utc(t0), to_zoned_utc(t1))
```

The generated layer accepts `ZonedDateTime` for time params (already UTC after `to_zoned_utc`) and `String`/`Date` for OWSMP params. Optional query params (`skip`, `take`, `filter`, `todate`, `shadow`) come through as keyword arguments.

## Key conventions worth remembering

- **One spec, two hosts.** Synthetic prefixes (`/publicationtool/core`, `/owsmp`) make the OpenAPI spec valid; the routing hook in `client.jl` strips them at request time. Touching one without the other breaks the wire format.
- **Anonymous PT, keyed OWSMP.** PT's `Authorization: Bearer …` is optional; OWSMP requires `AUTH_API_KEY: <key>`. The two clients embed the right scheme.
- **Times are `ZonedDateTime` UTC.** The generated layer accepts `ZonedDateTime`; `to_zoned_utc(DateTime|Date|ZonedDateTime)` does the coercion. Naive `DateTime` is treated as UTC (no implicit local-zone guess).
- **`200 OK` ≠ data exists.** PT will happily return `{"data":[]}` for empty windows. OWSMP returns an empty array. Check the response shape before assuming records.
- **Response shape varies per endpoint.** `DomainEnvelope` (`finalComputation` &c.) carries `totalRowsWithFilter` + `data`. `DataEnvelope` (`netPos`, `monitoring`, …) carries `data` only. OWSMP returns bare arrays of free-form objects.
- **`application/problem+json` on errors.** PT's ASP.NET Core middleware returns RFC 7807 problem details (4xx/5xx). The spec maps them to `ProblemDetails`; reach for `resp.raw` on the `ApiResponse` to see the original body if you need it.
- **Rate limit on OWSMP: 100 req/min.** Wrap with `with_defaults` + a `TokenBucket` if you batch-pull.
- **Reliability middleware composes.** Use `with_defaults(...) do ... end` to wrap any call in retry → rate-limit → timeout, with logging on the outside. `RetryPolicy()` defaults retry on 408/429/5xx.

## Common commands

```bash
# Rebuild spec → re-run codegen end to end:
julia scripts/build_openapi.jl
julia --project gen/regenerate.jl

# Full test suite (Aqua + JET when installed)
julia --project -e 'using Pkg; Pkg.test()'

# Skip the linting (Aqua/JET) suite — fastest local loop
OPENAPI_SKIP_LINTING=1 julia --project -e 'using Pkg; Pkg.test()'

# Pre-commit hooks (runic formatter, typos, ExplicitImports, markdownlint, yamlfmt, …)
pre-commit run --all-files   # or `prek` if installed
```

Codegen (Java 11+ and Node 18+) is a *maintainer* prerequisite. End users never need either — `src/api/` is committed plain Julia.

## When in doubt

- Need an endpoint's parameter list and meaning? `scripts/build_openapi.jl#ENDPOINT_TABLE` is the source of truth. Each row carries the path, params, summary, and a description.
- Need to know what `gen/regenerate.jl` will overwrite? Only `src/api/` — never `src/JAOEU.jl`, `src/client/`, `src/conveniences/`, `test/`, or any of the docs.
- Adding a new named wrapper? Mirror an existing one in `src/conveniences/queries.jl`: route through the right `*_api(client)` constructor, coerce times via `to_zoned_utc`, return the raw envelope so callers can unwrap with `rows`. Export it in `conveniences/conveniences.jl`.
- Adding a new endpoint? Edit `ENDPOINT_TABLE` in `scripts/build_openapi.jl`, run that script, then `gen/regenerate.jl`. A new Julia function will appear under `JAOEU.JAOEUAPI`.
